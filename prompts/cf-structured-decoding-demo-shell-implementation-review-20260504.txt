We are building Continuous Function, a Next.js/React learning platform for deep mathematical ideas. The product goal is to make learning feel joyful, deep, rigorous, visual, and worth returning to: learners should move from intuition to math to code to manipulation without losing the central invariant.

This is the implementation review for the Structured Decoding demo-shell rollout slice selected by Oracle/GPT Pro.

Oracle selected this slice with the central learner action:
- At the current DFA state q, predict whether the raw highest-logit token survives the schema mask M(q), gets replaced by the highest valid token, or has no valid continuation.
- Reveal should show raw logits -> schema mask -> masked/renormalized probabilities -> next token/state.
- The guarantee boundary must remain explicit: formal schema membership is not truth, safety, or task success.

What I changed:
- `content/domains/llm-systems/concepts/structured-decoding/viz.tsx`
  - Replaced the direct foundation export with a `VizShell` + `VizStageAdapter` route wrapper.
  - The shell challenge asks the learner to predict raw-top vs masked-winner vs impossible continuation.
  - Notes caveat that this toy is an ordered token-level schema and production systems work over real tokenizer IDs/richer grammars.
- `components/foundations/StructuredDecodingViz.tsx`
  - Added `chrome?: 'legacy' | 'notebook'` and `conceptId` props, defaulting to legacy behavior.
  - Added notebook-only prediction state and reveal gating.
  - Pre-reveal: shows current state, prefix, raw logits/probabilities, and prediction options, but hides allowed-token answers, masked probabilities, result diagnosis, and masked winner in demo-state.
  - Reveal: shows allowed column, masked probabilities, rejection mass, masked top token, next token/state, correctness, and the formal-guarantee boundary.
  - Added controls for schema mask on/off, model profile, temperature, and top-p-after-mask.
  - Added notebook actions for reveal, emit revealed token, run to EOS, and reset.
  - Mask off + format-confused can emit `"DROP"` and break the prefix.
  - Semantic mismatch + mask on can run to EOS and accept while selecting the wrong source.
  - Emits compact demo-state only in notebook mode.
  - Added accessible prediction buttons with `aria-pressed`, polite reveal feedback, visible focus states, and a local scroll token table for mobile.
- `content/domains/llm-systems/concepts/structured-decoding/content.mdx`
  - Updated the Interactive Demo section to match the prediction/reveal task.
- `content/domains/llm-systems/concepts/structured-decoding/concept.yaml`
  - Updated the short description/date to include the guarantee boundary.
