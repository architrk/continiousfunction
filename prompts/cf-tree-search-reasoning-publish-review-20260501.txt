I rewrote the page to avoid making PUCT, UCT, visit-count MCTS, AlphaZero-style search, agents, tool use, or product histories the teaching core.

The page teaches:
- complete-trace best-of-N spends budget on full solutions; tree search spends budget on partial reasoning prefixes
- prefix states h=(a_1,...,a_t), next-step children ha, local verifier scores r_phi(h,a), and hidden terminal correctness u(z)
- a visible subtree T_t and frontier F_t
- cumulative local score G_phi(h)
- a deterministic best-first expansion rule h_t = argmax_h [G_phi(h)+b_phi(h)]
- a max backup V_t(h) over the visible tree
- selected visible path via backed-up child values
- a compact cost model for revealed children
- the limitation that verifier errors can be amplified by the backup

Interactive demo:
- title: Prefix Budget Explorer
- fixed algebra task: 2(x+3)=14
- controls: clean/noisy verifier mode, 0..6 prefix expansions after root, show hidden correctness toggle
- clean mode default budget 2 selects root -> C -> C1 -> C1a, a correct terminal trace
- noisy mode with the same budget selects root -> A -> A1, a wrong terminal trace, because the invalid A branch has high proxy scores
- metrics: selected visible path, selected terminal, expanded prefixes, frontier size, V(root)
- state view: unseen, frontier, expanded, selected/backed-up path, terminal nodes
- hidden correctness is diagnostic only and does not affect selection
