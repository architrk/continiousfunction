Claim under review:
"For one executed differentiable program, a computation graph uses elementary-operation nodes and dependency edges, stores each reused intermediate node value once in the forward pass, and makes reverse-mode/backprop local: downstream sensitivities from every use accumulate at a node before being sent to its inputs."

Cited source:
Baydin et al., "Automatic Differentiation in Machine Learning: a Survey", JMLR 18(153):1-43 (2018), https://jmlr.csail.mit.edu/papers/v18/17-468.html

Primary-source evidence I checked before this prompt:
- JMLR PDF lines 472-494: evaluation traces and computational graphs are used to visualize dependency relations between intermediate variables; AD works from a numeric evaluation trace with input, intermediate, and output values needed for derivatives by chain-rule composition.
- JMLR PDF lines 487-489: Figure 4 is a computational graph for an example function with intermediate variables.
- JMLR PDF lines 727-738: reverse mode is a generalized backpropagation algorithm; it runs code forward, populates intermediate variables, records graph dependencies by bookkeeping, then propagates adjoints backward.
- JMLR PDF lines 745-783: a reused variable's output sensitivity is the sum of downstream contributions, computed incrementally, and the reverse pass starts from output adjoint 1 to obtain input derivatives in one reverse pass.
- JMLR PDF lines 1061-1068: in neural-network training, backpropagation is a special case of AD applying reverse mode to a network error objective to compute partial derivatives for weight updates.
- JMLR PDF lines 1082-1097: static and dynamic computational graph terminology distinguishes prebuilt graph execution from define-by-run graphs constructed on the fly from regular program execution.
