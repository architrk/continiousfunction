Primary-source evidence I extracted from the Dao et al. paper:
- Page 1 title/abstract frames FlashAttention as exact attention with IO-awareness. The abstract says it is an IO-aware exact attention algorithm using tiling to reduce reads/writes between GPU HBM and GPU on-chip SRAM, and analyzes fewer HBM accesses than standard attention.
- Figure 1 caption says FlashAttention uses tiling to prevent materialization of the large N x N attention matrix on slow GPU HBM. It describes blocks of K and V loaded to SRAM, blocks of Q loaded to SRAM, and output written back to HBM.
- Page 2 text says FlashAttention computes exact attention with far fewer memory accesses, with the main goal of avoiding reads/writes of the attention matrix to/from HBM. It also says it does not read/write the large N x N attention matrix to HBM.
- Page 4 standard attention algorithm explicitly writes S=QK^T and P=softmax(S) to HBM. The paper then says it computes exact attention with fewer HBM reads/writes and without storing large intermediate matrices.
- Page 4/5 section 3.1 describes tiling: split Q,K,V into blocks, load them from slow HBM to fast SRAM, and use a modified softmax reduction/tiling approach. Algorithm 1 maintains O, l, and m and updates block statistics.
- Page 6 Theorem 2 states standard attention requires Theta(Nd + N^2) HBM accesses while FlashAttention requires Theta(N^2 d^2 M^-1) HBM accesses under stated SRAM-size conditions; the paper says this is many times fewer HBM accesses.
