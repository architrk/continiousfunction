This review is for one concrete P0 retrofit:

Variational Autoencoders now has a prediction-first ELBO gap reveal. The learner can adjust decoder mode, observed x, q mean, and q std. Before reveal, they see the prior, normalized likelihood shape, q(z|x), and a hidden ELBO identity shell. They must predict whether the hidden gap diagnoses:

1. Tight bound
2. Shift or scale mismatch
3. Family mismatch

After reveal, the UI shows the true posterior curve, log p(x), ELBO, gap, KL(q || posterior), identity error, reconstruction term, KL(q || prior), post-reveal explanation, and quick alignment helpers. The demo emits measured state only after reveal. Any mode/x/q/prediction change after reveal should clear the shared demo state, hide the answer again, and require a fresh prediction.

The earlier Oracle scope response is attached. It selected VAE as the best next slice after Random Variables and warned against positive-vs-negative mode predictions because the quadratic decoder can have symmetric co-winners.

What I changed:

- `content/domains/generative-models/concepts/vaes/viz.tsx`
  - Adds `emitDemoState` / `clearDemoState`.
  - Adds a gated prediction panel.
  - Hides true posterior and ELBO diagnostics before reveal.
  - Classifies the expected diagnosis with:
    - tight if `gap <= 0.05`
    - family mismatch if quadratic mode and `xObs >= 0.65`
    - otherwise shift/scale mismatch
  - Emits compact measured state after reveal.
  - Clears shared demo state on mount, unmount, control changes, prediction changes, and quick actions.

- `content/domains/generative-models/concepts/vaes/content.mdx`
  - Rewrites the Interactive Demo copy so it does not pre-announce posterior/gap answers or post-reveal helper buttons.

- `components/concepts/VAEsViz.test.tsx`
  - Adds focused regressions for pre-reveal hiding, reveal requiring prediction, quadratic family-mismatch state emission, stale-state clearing after q changes, no positive/negative cause prediction choices, and MDX leak phrases.
