Project: Continuous Function, a Next.js/TypeScript learning platform for deep, joyful, mechanism-first learning. This repo is intentionally staying static/export-safe until auth/database setup is real. The private branch is private/research-learning-loop-20260505.

Prior Oracle/GPT Pro scope selected the next bounded slice: local-object-next-action-journal-v1. The intended behavior is:
- In ResearchReadingRoom, a selected object with a valid anchor.objectKey can save one bounded browser-local draft note and one bounded next action.
- The draft is keyed only by ContentObjectKey, never by href, title, discussionAnchorId, slug, or list index.
- The draft persists in localStorage under cf:object-action-journal, refreshes via storage/custom events, reappears only for the same selected object, and can be cleared without touching other object drafts.
- The selected object's grounded AI handoff prompt includes the matching local draft.
- Objects without objectKey show a disabled state and cannot save.
- The copy must not imply account sync; it should say local browser only.
- No Clerk, Neon, API route, middleware, dynamic server behavior, database writes, AI provider calls, account/profile/dashboard, or live collaboration should be introduced.
