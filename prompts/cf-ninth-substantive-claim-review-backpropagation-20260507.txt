Claim under review:
"Backpropagation computes neural-network gradients by storing needed forward-pass values, then applying the chain rule backward from a scalar loss: each layer receives a downstream sensitivity, combines it with local derivatives/VJPs, and reuses that signal to form parameter gradients in one reverse pass."

Cited sources:
1. Rumelhart, Hinton, and Williams, "Learning representations by back-propagating errors", Nature 323, 533-536 (1986), https://www.nature.com/articles/323533a0
2. Baydin et al., "Automatic Differentiation in Machine Learning: a Survey", JMLR 18(153):1-43 (2018), https://jmlr.csail.mit.edu/papers/v18/17-468.html

Primary-source evidence I checked before this prompt:
- Nature page lines 42-44: the official abstract describes back-propagation for networks of neurone-like units and repeated weight adjustments to minimize the difference between actual and desired outputs.
- Nature page lines 153-165: official citation metadata for the Rumelhart/Hinton/Williams Nature paper.
- JMLR PDF lines 85-93: backpropagation is described as gradient descent in neural-network weight space, using backward propagation of objective sensitivity and the chain rule to compute partial derivatives of the objective with respect to each weight; Baydin et al. also state it is essentially reverse-mode AD on the network evaluation composed with the objective.
- JMLR PDF lines 138-151: Figure 1 says training inputs are fed forward, error is computed, then the error adjoint is propagated backward to give weight gradients used by gradient descent; input gradients can be computed in the same backward pass.
- JMLR PDF lines 727-738: reverse mode propagates derivatives backward from an output; for backpropagation the output is scalar error; the first phase runs code forward and records graph dependencies, the second propagates adjoints backward.
- JMLR PDF lines 740-783: the reverse pass accumulates local chain-rule contributions and starts from output adjoint 1.
- JMLR PDF lines 788-804: for f: R^n -> R one reverse-mode application computes the full gradient, which is why reverse mode is the mainstay in the form of backpropagation for scalar objectives with many parameters.
- JMLR PDF lines 870-912: reverse mode computes transposed Jacobian-vector products by initializing the reverse phase with an output vector.
- JMLR PDF lines 913-917: reverse mode has increased storage requirements, supporting the "saved forward values / bookkeeping" caveat.
