Claim under review:
"Reverse-mode AD records a forward evaluation trace of primitive operations and saved values, seeds a scalar output cotangent with 1, then sweeps backward applying local VJPs and += accumulation so one reverse pass computes the full gradient of one scalar loss with respect to many inputs."

Cited source:
Baydin et al., "Automatic Differentiation in Machine Learning: a Survey", JMLR 18(153):1-43 (2018), https://jmlr.csail.mit.edu/papers/v18/17-468.html

Primary-source evidence I checked before this prompt from the official JMLR PDF:
- PDF page 3: Baydin et al. distinguish AD from symbolic/numerical differentiation and say AD evaluates derivatives accurately by applying symbolic differentiation rules while keeping track of derivative values.
- PDF page 12: Section 3.2 says reverse mode AD propagates derivatives backward from a given output, complements each intermediate variable vi with an adjoint/cotangent, and for backpropagation the considered output is a scalar error.
- PDF page 12: Baydin et al. state reverse mode uses a two-phase process: run original function code forward to populate intermediate variables and record graph dependencies, then calculate derivatives by propagating adjoints backward from outputs to inputs.
- PDF page 12: In the reused-variable example, a variable affects the output through two downstream variables, so its adjoint is a sum of downstream contributions; the paper shows the contribution being computed incrementally.
- PDF pages 12-13: The reverse pass starts with output adjoint v5_bar = y_bar = partial y / partial y = 1 and computes both input derivatives in one reverse pass.
- PDF page 12: For f:R^n -> R, one reverse-mode application computes the full gradient, while forward mode needs n passes; this is why reverse mode is the mainstay for scalar objectives with many parameters.
- PDF page 13: Baydin et al. describe reverse mode computing transposed Jacobian-vector products by initializing the reverse phase with an output vector.
- PDF page 13: Baydin et al. note reverse mode has increased storage requirements and mention checkpointing/data-flow analysis as implementation strategies.
- PDF page 18: Baydin et al. distinguish static and dynamic computational graphs, useful for caveat wording around control-flow conventions.
