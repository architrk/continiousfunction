Claim under review:
"At incremental decoding time, KV cache reads can become a memory-bandwidth bottleneck; sharing key/value heads with MQA or GQA reduces KV tensor size while preserving multiple query heads."

Cited sources:
- Shazeer, "Fast Transformer Decoding: One Write-Head is All You Need" (2019), https://arxiv.org/pdf/1911.02150
- Ainslie et al., "GQA: Training Generalized Multi-Query Transformer Models from Multi-Head Checkpoints" (EMNLP 2023), https://aclanthology.org/2023.emnlp-main.298.pdf

Primary-source evidence I checked before this prompt:
- Shazeer PDF page 1: the abstract says incremental inference is often slow due to memory-bandwidth cost from repeatedly loading large keys and values tensors; MQA shares keys and values across attention heads, reducing tensor size and incremental-decoding memory bandwidth requirements.
- Shazeer PDF page 1: the introduction says incremental Transformer inference speed is limited by memory bandwidth needed to reload large keys and values tensors encoding attention-layer state.
- Shazeer PDF page 4: the analysis says the memory-to-compute ratio can make memory bandwidth a major bottleneck; it ties the hard term to reloading K and V tensors and proposes reducing the K/V tensor size by removing their heads dimension while maintaining the heads dimension in queries.
- Shazeer PDF page 5: Multi-query attention is identical to multi-head attention except the different heads share a single set of keys and values.
- Shazeer PDF page 7: the experiment reports much faster incremental decoder step times for the multi-query model than the baseline in the tested setting.
- Ainslie et al. PDF page 1: the abstract says MQA uses a single key-value head and drastically speeds decoder inference, while GQA uses an intermediate number of key-value heads and achieves quality close to MHA with comparable speed to MQA.
- Ainslie et al. PDF page 1: the introduction says autoregressive decoder inference is bottlenecked by memory bandwidth from loading decoder weights and all attention keys/values at every step, and that loading keys/values can be sharply reduced through MQA, which uses multiple query heads but single key/value heads.
- Ainslie et al. PDF page 2: Figure 2/Section 2.2 says MHA has H query/key/value heads, MQA shares single key/value heads across all query heads, and GQA shares single key/value heads for each group of query heads, interpolating between MHA and MQA.
- Ainslie et al. PDF page 2: Section 2.2 says reducing H key/value heads to one reduces key-value cache size and the amount of data loaded by a factor of H; GQA keeps a proportional decrease in bandwidth and capacity as model size increases.
- Ainslie et al. PDF page 3: Table 1 and main results show GQA-8-XXL close to MHA-XXL quality with speed close to MQA in their tested setup.
