We are building Continuous Function, a static-export-friendly Next.js research-learning platform. This checkpoint implements the first object-attached discussion model.

Goal:
- Preserve stable discussion identity for concept, equation, paper, visualization, toy-experiment, claim, and misconception objects.
- Keep the feature as a local/static placeholder only: no backend, no accounts, no Discourse integration, no fake replies, no localStorage discussion history.
- Replace scattered disabled "Open thread" style affordances with work-focused anchor cards that say the question is attached to the object and the thread is not live yet.

Implementation summary:
- Added `lib/discussionAnchors.ts` with anchor/thread versions, object/surface constants, path-like anchor ID builder, DOM ID helper, internal/external URL validators, anchor guards, thread guards, and small builders.
- Added unit tests in `lib/discussionAnchors.test.ts`.
- Added reusable `components/discussion/DiscussionAnchorCard.tsx`, `DiscussionAnchorList.tsx`, and index export.
- Added `components/discussion/DiscussionAnchorCard.test.tsx`, though the component Jest project appears to hang in this environment even for an existing component test.
- Updated `lib/learningRouteSnapshot.ts` so saved source objects can carry new object types and optional `discussionAnchorId`, while keeping legacy `lab`, `thread`, and `question` readable.
- Updated `components/product/AttentionServingModule.tsx` to replace local discussion objects with typed discussion seeds rendered by the shared list.
- Updated `components/product/PaperConceptMapper.tsx` to add explicit IDs to claims/equations and render paper, claim, equation, and toy-lab discussion anchors from code-native objects.
- Updated `components/product/GraphProductNavigator.tsx` to stop creating new `thread` objects, use claim/misconception/toy-experiment objects with IDs/prompts, persist `discussionAnchorId` on source objects, and render shared anchor cards.
- Updated `components/concepts/ConceptNotebookPage.tsx` to add concept-level and visualization-level placeholder anchors.
