You are GPT Pro acting as a senior product designer, learning scientist, mathematical visualization designer, and ruthless UX critic for Continuous Function.

Context:
- Continuous Function is an AI-first learning platform for the mathematics behind modern deep learning and frontier AI.
- The desired product feeling is "Living Notebook Lab": part atlas, part notebook, part lab, part paper-reading room, part careful AI tutor.
- Pedagogical loop: Question -> Object -> Prediction -> Manipulation -> Evidence -> Invariant -> Next Move.
- The user feels the generated image direction boards are much richer than the current live app, and that the app experience is considerably inferior.
- Your job is to explain that gap and convert it into buildable guidance.

Important repo principles:
- Images are implementation-facing direction boards, not decoration.
- Repo-native UI and demos should carry precise learning mechanisms.
- The app must remain high-signal, rigorous, source-grounded, and not become a marketing site or generic dashboard.
- Current app has some strong pieces: route strip, concept notebook, Paper Mapper, graph, search, /me study memory, adaptive loop contract, object rooms.
- The issue is that the experience still often feels like assembled panels and dense text, not a cohesive immersive learning instrument.

Live app tour observations from 2026-05-24:
1. Homepage command deck works, but the first viewport still reads more like an explanatory product panel than an irresistible learning instrument.
2. Domains page gives route-aware entry points and role lenses; useful, but visually still too card-grid / admin-like.
3. Efficient Attention concept notebook has selected objects, role lenses, Predict/Code/Ask, object room, and source cards. It is functionally rich but visually dense; the object room is powerful but not yet spatially memorable.
4. Paper Mapper converts a pasted paper clue into source check, equations, source spans, claims, role lenses, and saved route. It is one of the best product ideas, but it still feels like a form plus panels rather than a magical research-to-learning cockpit.
5. Graph carries the current investigation and route state. It is useful but does not yet feel like a living map that teaches the next move.
6. Search is route-aware and buckets results into repair/inspect/witness/carry actions. Good functionally; visual hierarchy still does not make it feel as special as the idea.
7. /me Study Memory shows durable memory, adaptive next move, and server contract checks. Honest, but it feels like developer-contract preview rather than a learner’s personal study space.
8. Attention Path is the strongest end-to-end module. It carries paper clue, equation, serving variables, prediction lab, and route. It should become the visual model for other surfaces.
9. Legacy Foundations is broad and useful, but visually/migrationally inconsistent with the newer notebook language.
10. Vision/Pillars state the philosophy, but the actual first-use experience still does not fully embody it.

Generated image study already extracted these recurring patterns:
- Persistent learning shell: stable spine, current route/progress, one clear workspace.
- Adaptive cockpit: learner/researcher/experimenter/professor lenses change immediate question and next move.
- Object-attached learning: questions, source checks, code witnesses, demos, misconceptions, and comments orbit exact objects.
- Evidence memory: compact portable observations: what changed, what stayed invariant, what to test next.
- Paper-to-atlas workflow: paper spans become objects, equations, demos, caveats, and routes.
- Research commons: source-grounded discussion with AI focused on evidence, assumptions, summaries, experiments.
- Device continuity: mobile preserves current object and next move.
