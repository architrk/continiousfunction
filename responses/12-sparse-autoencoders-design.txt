## 2) Math core (2–3 key equations)

Let (x \in \mathbb{R}^d) be a model activation vector (often **residual stream** at a layer/site).

**(1) SAE encode → sparse latents, decode → reconstruction**
[
z = \mathrm{ReLU}(W_{\text{enc}}(x-b_{\text{pre}})+b_{\text{enc}}),\qquad
\hat x = W_{\text{dec}} z + b_{\text{pre}}
]
This makes the key idea concrete: **a sparse code (z)** explains the dense activation (x) via a learned dictionary (W_{\text{dec}}). 

**(2) Classic SAE objective (reconstruction + sparsity penalty)**
[
\mathcal L = \lVert x-\hat x\rVert_2^2 + \lambda \lVert z\rVert_1
]
Sparsity is what pushes the model toward **monosemantic, human-nameable latents** (instead of dense uninterpretable factors). 

**(3) Modern “Top‑K / k‑sparse” variant (directly control sparsity)**
[
z = \mathrm{TopK}(W_{\text{enc}}(x-b_{\text{pre}}), k),\qquad
\mathcal L = \lVert x-\hat x\rVert_2^2
]
This replaces “tune (\lambda)” with “set exactly **k active features per token**,” improving the reconstruction–sparsity frontier and simplifying scaling.

Why frontier cares:

* You can’t reliably “neuron-interpret” huge models due to **polysemanticity**; SAEs give a scalable substitute unit: **feature latents**.

## 4) Missing intuition to teach

This is the “aha” layer your users likely don’t have yet:

1. **SAE = “learn a parts dictionary for the residual stream.”**
   Each column of (W_{\text{dec}}) is a candidate *feature direction*. The sparse vector (z) says “which parts are present right now, and how strongly.”

2. **Sparsity is the interpretability prior.**
   A dense code can represent anything but names nothing. Sparse codes force the model to reuse a small set of directions across many examples—making “this direction” show up repeatedly in similar semantic contexts.

3. **The real knob isn’t “reconstruction error,” it’s “explanatory budget per token.”**
   In k‑sparse SAEs, (k) literally sets: *how many features are allowed to explain an activation*. That’s an intuitive tradeoff slider users can grasp immediately. 

4. **SAE pathologies are not side notes—they’re the whole game at scale.**

   * **Shrinkage bias** from L1 can understate feature magnitudes; gated approaches explicitly address this. 
   * **Dead latents** (features that never activate) become severe at scale; modern recipes include explicit mitigations. 

5. **Interpretability becomes actionable when features become causal handles.**
   Once you have features, you can do: ablation, amplification, causal tracing, circuit extraction—turning “interpretability” into “debugging & control.”

## 5) 2–3 canonical papers (2024–2025)

2. **Rajamanoharan et al., “Improving Dictionary Learning with Gated Sparse Autoencoders” (2024)** — addresses L1 shrinkage by separating “which features fire” from “how strongly,” improving the pareto frontier. 
3. **Marks et al., “Sparse Feature Circuits: Discovering and Editing Interpretable Causal Graphs in Language Models” (2024)** — uses SAE features as units for circuit discovery and editing, enabling scalable pipelines beyond neuron/head circuits. ([arXiv][3])
