title: "Overparameterization & Generalization (Double Descent)"
---

## Intuition

Classical advice says: if your model is too big, it will overfit.

Modern practice says: make it even bigger, and it might get *better* again.

Double descent is the empirical pattern behind that contradiction:

1. As capacity increases, test error initially falls (bias decreases).
2. Near the **interpolation threshold** (where training error hits ~0), test error can spike.
3. Past that, as capacity keeps growing, test error often falls again.

The key idea is that in the overparameterized regime there are many solutions that fit the training data perfectly. Optimization does not pick one at random: it has an **implicit bias** (often toward low-norm or "simple" solutions), and those can generalize surprisingly well.

Grokking is a related phenomenon: a model can fit training data early (memorize), then only much later transition into a cleaner generalizing solution.

## Math

In a linear regression view, let $X\\in\\mathbb R^{n\\times d}$ be features and $y\\in\\mathbb R^n$ targets.

- If $d < n$ and $X$ has full column rank, the least-squares solution is unique:

$$\\hat w = \\arg\\min_w \\|Xw - y\\|_2^2.$$

- If $d > n$, there are infinitely many interpolating solutions with $Xw=y$.

A common implicit bias (e.g., gradient descent from small initialization in linear models) is the **minimum-norm interpolant**:

$$\\hat w_{\\min\\,\\|w\\|} = X^\\top (XX^\\top)^{-1} y.$$

As $d$ crosses $n$, the geometry of $X$ changes, and generalization can exhibit a peak near interpolation followed by a second descent as the minimum-norm solution becomes "simpler" relative to the space of all interpolants.

## Code

```python
import numpy as np

rs = np.random.RandomState(0)
n_train, n_test = 80, 2000
noise = 0.2

def min_norm_fit(X, y):
    # w = X^T (X X^T)^{-1} y  (works when d >= n and X X^T is invertible)
    return X.T @ np.linalg.solve(X @ X.T + 1e-8 * np.eye(X.shape[0]), y)

for d in [10, 30, 60, 80, 100, 150, 300]:
    w_true = rs.randn(d) / np.sqrt(d)
    Xtr = rs.randn(n_train, d)
    ytr = Xtr @ w_true + noise * rs.randn(n_train)
    Xte = rs.randn(n_test, d)
    yte = Xte @ w_true + noise * rs.randn(n_test)

    w_hat = min_norm_fit(Xtr, ytr)
    train_mse = float(np.mean((Xtr @ w_hat - ytr) ** 2))
    test_mse = float(np.mean((Xte @ w_hat - yte) ** 2))
    print("d =", f"{d:>3}", "train_mse =", round(train_mse, 3), "test_mse =", round(test_mse, 3))
```
