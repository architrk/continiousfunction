## Intuition

KL divergence answers a simple question:

If the world is generated by a distribution $p$, but you *act as if* it were generated by $q$, how much do you pay for that mismatch?

One good mental model is **extra code length**. If you build an optimal compressor for $q$ and then feed it samples from $p$, you will use more bits than necessary. KL is the expected overhead.

Another mental model is **"surprise regret"**: KL is the average amount by which your log-probabilities are worse under $q$ than under the truth $p$.

The direction matters:

- $\\mathrm{KL}(p\\|q)$ is "data vs model" and blows up if the model assigns near-zero probability to things that happen in data.
- $\\mathrm{KL}(q\\|p)$ is "model vs reference" and heavily penalizes putting mass where the reference thinks events are extremely unlikely (this is the flavor you see in KL-regularized RLHF).

## Math

For a discrete random variable $X$, the KL divergence from $p$ to $q$ is:

$$\\mathrm{KL}(p\\|q) := \\sum_x p(x)\\,\\log\\frac{p(x)}{q(x)}.$$

Equivalently, as an expectation:

$$\\mathrm{KL}(p\\|q) = \\mathbb E_{x\\sim p}\\,[\\log p(x) - \\log q(x)].$$

For a continuous variable, replace the sum by an integral.

Key properties:

- **Non-negativity (Gibbs inequality):** $\\mathrm{KL}(p\\|q) \\ge 0$, with equality iff $p=q$ almost everywhere.
- **Asymmetry:** generally $\\mathrm{KL}(p\\|q) \\ne \\mathrm{KL}(q\\|p)$.
- **Cross-entropy identity:** define entropy $H(p) := -\\sum_x p(x)\\log p(x)$ and cross-entropy $H(p,q) := -\\sum_x p(x)\\log q(x)$. Then

$$H(p,q) = H(p) + \\mathrm{KL}(p\\|q).$$

This is why maximum likelihood (minimizing cross-entropy) is equivalent to minimizing $\\mathrm{KL}(p_{\\text{data}}\\|p_\\theta)$ up to a constant $H(p_{\\text{data}})$ that does not depend on the model.

## Code

```python
import numpy as np

def kl(p, q, eps=1e-12):
    p = np.clip(np.asarray(p, dtype=float), eps, 1.0); p = p / p.sum()
    q = np.clip(np.asarray(q, dtype=float), eps, 1.0); q = q / q.sum()
    return float(np.sum(p * (np.log(p) - np.log(q))))

p = [0.55, 0.25, 0.15, 0.05]              # "data"
q = [0.80, 0.15, 0.04, 0.01]              # "model"

kl_pq = kl(p, q)
kl_qp = kl(q, p)
H_p = -np.sum(np.array(p) * np.log(np.array(p)))
H_pq = -np.sum(np.array(p) * np.log(np.array(q)))

print("KL(p||q):", round(kl_pq, 4), "  KL(q||p):", round(kl_qp, 4))
print("H(p,q):", round(float(H_pq), 4), "  H(p)+KL(p||q):", round(float(H_p + kl_pq), 4))
```
