## Intuition

The learning rate is the step size of training. It is also one of the most sensitive knobs in deep learning: the same model can diverge, crawl, or converge depending on the schedule.

Two recurring patterns in large-model training:

- **Warmup:** start small, then ramp up. Early gradients are noisy and optimizers like Adam have biased moment estimates at the start. Warmup avoids "blowing up" before the optimizer settles.
- **Decay:** once the model has found a good region, reduce the step size so you can refine instead of bouncing around.

Modern LLM training commonly uses: warmup -> (optional plateau) -> cosine decay.

## Math

Let $\\eta_t$ be the learning rate at step $t$.

**Linear warmup** to a peak $\\eta_{\\max}$ over $T_w$ steps:

$$\\eta_t = \\eta_{\\max}\\cdot \\frac{t}{T_w},\\qquad t\\le T_w.$$

**Cosine decay** from $\\eta_{\\max}$ to $\\eta_{\\min}$ over $T$ total steps (after warmup):

$$\\eta_t = \\eta_{\\min} + \\tfrac12(\\eta_{\\max}-\\eta_{\\min})\\left(1+\\cos\\left(\\pi\\,\\frac{t-T_w}{T-T_w}\\right)\\right),\\qquad t\\ge T_w.$$

A simple classical alternative is:

$$\\eta_t = \\frac{\\eta_0}{\\sqrt{t+1}}.$$

The schedule interacts with curvature and sharpness: if the learning rate is too high relative to local curvature, training becomes unstable (edge-of-stability behavior).

## Code

```python
import numpy as np

def warmup_cosine(T, Tw, eta_max, eta_min):
    lr = np.zeros(T)
    for t in range(T):
        if t < Tw:
            lr[t] = eta_max * (t + 1) / max(1, Tw)
        else:
            u = (t - Tw) / max(1, T - Tw - 1)
            lr[t] = eta_min + 0.5 * (eta_max - eta_min) * (1 + np.cos(np.pi * u))
    return lr

lr = warmup_cosine(T=10000, Tw=500, eta_max=3e-4, eta_min=3e-5)
print("lr[0], lr[Tw], lr[-1]:", lr[0], lr[500], lr[-1])
print("avg lr:", float(lr.mean()))
```
