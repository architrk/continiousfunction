## Intuition

A function is a rule that turns an input into an output. You can think of it as a machine: put in $x$, get out $f(x)$.

Functions matter because they let us talk precisely about *dependencies*: how one quantity changes when another changes. Calculus is basically the study of those changes.

Two ideas show up constantly in ML:

- **composition:** one function feeding into another (layers in a network)
- **vector-valued functions:** many inputs, many outputs (models)

## Math

A function is written

$$
f: X \to Y
$$

meaning: for each input $x \in X$ (the **domain**), the output $f(x)$ lies in $Y$ (the **codomain**).

The **graph** of a real-valued function $f: \mathbb{R} \to \mathbb{R}$ is the set of points

$$
\{(x, f(x)) : x \in \mathbb{R}\}.
$$

**Composition** means chaining machines together. If $g: X \to Y$ and $f: Y \to Z$, then

$$
(f \circ g)(x) = f(g(x)).
$$

That “feed the output of $g$ into $f$” pattern is the exact shape of neural networks.

## Code

```python
import numpy as np

# A simple function
f = lambda x: x**2 + 1

# Another function
g = lambda x: np.sin(x)

# Composition: f(g(x))
x = np.linspace(-2, 2, 5)
print("x:", x)
print("g(x):", g(x))
print("f(g(x)):", f(g(x)))
```
