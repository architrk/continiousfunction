## Intuition

A model can only "think" in the space it uses internally. In deep learning, that internal space is usually a vector space: an **embedding**.

The point of representation learning is to make geometry meaningful. We want vectors where:

- things that "mean the same" are close,
- things that are different point in different directions,
- and simple operations (dot products, distances, averages) line up with useful questions (retrieval, clustering, analogy, control).

When this works, embeddings become an interface. You can do nearest-neighbor search, build a classifier with a linear layer, or steer behavior by moving along a direction. When it fails, vectors collapse into weird shapes (anisotropy, degenerate norms) and similarity becomes noisy.

## Math

Let an encoder map inputs to vectors:

$$f_\\theta : \\mathcal X \\to \\mathbb R^d, \\qquad z = f_\\theta(x).$$

### Similarity as a dot product (often cosine)

A common choice is cosine similarity, which is just a dot product of normalized vectors:

$$\\mathrm{sim}(z, z') = \\frac{z^\\top z'}{\\lVert z\\rVert_2\\,\\lVert z'\\rVert_2} = \\hat z^\\top \\hat z'.$$

Normalization matters because it makes similarity depend on **direction** instead of length.

### Contrastive learning (InfoNCE)

Suppose we have "positive pairs" $(x_i, y_i)$ (two views of the same thing: two crops of an image, two versions of a sentence, etc.). Encode them:

$$z_i = f_\\theta(x_i), \\qquad u_i = g_\\theta(y_i).$$

InfoNCE treats $(z_i, u_i)$ as the correct match among a batch of candidates:

$$\\mathcal L = -\\frac{1}{N} \\sum_{i=1}^N \\log \\frac{\\exp\\big(\\mathrm{sim}(z_i, u_i)/\\tau\\big)}{\\sum_{j=1}^N \\exp\\big(\\mathrm{sim}(z_i, u_j)/\\tau\\big)}.$$

- $\\tau > 0$ is the **temperature**: smaller $\\tau$ makes the softmax sharper (harder negatives, bigger gradients).
- This loss pushes positives to have higher similarity than negatives.

### A useful geometry sanity check: anisotropy

Embeddings often become **anisotropic** (many points bunch along a few dominant directions). A practical check is the covariance of normalized embeddings:

$$C = \\frac{1}{N}\\sum_{i=1}^N (\\hat z_i - \\bar z)(\\hat z_i - \\bar z)^\\top.$$

If a few eigenvalues dominate, your space has "dead directions" and cosine similarity can get biased. Normalization (LayerNorm/RMSNorm) and whitening-like tricks often change this geometry dramatically.

## Code

```python
import numpy as np

def normalize(x):
    return x / (np.linalg.norm(x, axis=1, keepdims=True) + 1e-9)

def info_nce(z1, z2, tau):
    logits = (z1 @ z2.T) / tau
    logits = logits - logits.max(axis=1, keepdims=True)
    p = np.exp(logits); p = p / p.sum(axis=1, keepdims=True)
    return (-np.log(np.diag(p))).mean(), p

rs = np.random.RandomState(0)
N, d = 8, 16
base = rs.randn(N, d)
z1 = normalize(base + 0.1 * rs.randn(N, d))
z2 = normalize(base + 0.1 * rs.randn(N, d))

for tau in [0.1, 0.5, 1.0]:
    loss, p = info_nce(z1, z2, tau)
    print("tau=", tau, "loss=", round(loss, 3), "avg P(match)=", round(np.diag(p).mean(), 3))
```
